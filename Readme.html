How to compile the Operative System.

 Back to Top

To compile the Operative System, you need first to install the NASM compiler, and download the KolarOS® Coralriff®  Source Code, it can be downloaded on the Official GitHub page.

Also in Linux, you need to install Make and Git too.


Windows:

To compile the Operative System just run the "build-windows.bat" file that is located in the tools folder in the Source Code.
(You need to open the 'build-windows.bat' file and edit the paths to where you have the Source Code saved.)


Linux:

To compile the Operative System just run the Terminal. And now, go to the path where you have the Source Code saved and type 'make'

(The 'kolar.bin' file works too, so if you want you can run it)

When the Source Code is compiled, just run the 'run.bat' file to run the compiled iso in QEMU!

Source Code content.

 Back to Top

Files:

src/boot.asm - Is the boot file, works for well, Boot the OS.
tools/build-windows.bat - Is for compile the OS in Windows.
tools/cygwin1.dll - It's a .DLL used by the 'mkisofs.exe' file to convert the compiled .bin file to a .iso file.
bin/kolar.bin - The compiled OS in the .bin format, is bootable.
bin/kolar.iso - The compiled OS in the .iso format, is bootable.
bin/kolar.flp - The compiled OS in the .flp format, i don't know if it is bootable.
bin/kolar.elf - The compiled OS in the .elf format, i don't know if it is bootable.
tools/mkisofs.exe - It's used to convert the compiled .bin file to a .iso file in Windows with the 'build-windows.bat' file.
tools/nasm - It's just nasm direct access.
run.bat - Is used for run the compiled .iso in QEMU.
Makefile - The makefile for compiling the OS with Make.

Folders:

bin - Is where the compiled files are saved.
src - Is where the src is saved.
docs - Just documentation.
tool - Is where the tools and things are saved.
